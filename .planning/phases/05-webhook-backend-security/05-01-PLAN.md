<task type="auto">
  <name>Task 1: Implement webhook lifecycle handlers for call_started, call_ended, and call_analyzed</name>
  <files>execution/backend/main.py</files>
  <action>
Replace the 3 TODO stubs in the retell_webhook() function with full implementations. Add the DISCONNECT_TO_STATUS mapping constant and a helper to resolve disconnect reasons to lead statuses. Add `from datetime import datetime, timezone` to imports.

**DISCONNECT_TO_STATUS constant** (place above the endpoint definitions):
A dict mapping Retell's 31 disconnect_reason values to lead status strings or None (meaning "check if tool already set status"). Use the exact mapping from RESEARCH.md's "Disconnect Reason to Lead Status Mapping" code example. Key mappings: dial_no_answer -> "no_answer", dial_busy -> "busy", voicemail_reached -> "voicemail", ivr_reached -> "voicemail", error_* -> "failed", user_hangup/agent_hangup/inactivity/max_duration_reached -> None.

**call_started handler:**
- Extract lead_id from call_data.get("metadata", {}).get("lead_id")
- If lead_id exists, UPDATE leads SET status='in_call', last_call_at=datetime.now(timezone.utc).isoformat(), last_call_id=call_id WHERE id=lead_id AND status='calling'
- The .eq("status", "calling") makes it idempotent (skip if already in_call)
- Log the event

**call_ended handler:**
- Extract: lead_id from metadata, duration_ms, recording_url, transcript, from_number, to_number, disconnection_reason, call_status, start_timestamp, end_timestamp
- Compute duration_seconds = (duration_ms or 0) / 1000
- Compute ended_at = datetime.now(timezone.utc).isoformat()
- UPSERT call_logs on retell_call_id conflict with: lead_id, duration_seconds, recording_url, transcript, ended_at, from_number, to_number, disconnection_reason
- Use supabase.table("call_logs").upsert({...}, on_conflict="retell_call_id").execute()
- If Supabase Python client does not support on_conflict keyword, fall back to: try upsert({...}).execute(), if that fails try insert then update pattern
- After UPSERT, resolve lead status from disconnect reason:
  - Look up disconnection_reason in DISCONNECT_TO_STATUS
  - If mapped status is not None (dial failure / error), UPDATE leads SET status=mapped_status WHERE id=lead_id
  - If mapped status is None (user_hangup, agent_hangup, etc.), check if the call_logs row has an outcome field set (meaning log_call_outcome tool already ran). If outcome exists, skip. If no outcome exists and lead is still 'in_call', set lead status to 'declined' (fallback per CONTEXT.md).
- Log the event with disconnect reason

**call_analyzed handler:**
- Extract call_analysis from call_data.get("call_analysis", {})
- Extract: call_summary, user_sentiment, in_voicemail, call_successful, custom_analysis_data
- UPDATE call_logs SET call_summary=call_summary, sentiment=user_sentiment WHERE retell_call_id=call_id
- Only update if we have data (skip if call_analysis is empty)
- Log the event

**Important notes:**
- Use datetime.now(timezone.utc).isoformat() for timestamps, NOT literal "now()" string (Supabase client sends values as-is, not as SQL expressions -- per RESEARCH.md Pitfall 6)
- Keep each handler fast (single DB operation where possible) -- Retell retries after 10 seconds
- Wrap each handler's DB operations in try/except, log errors but still return {"status": "ok"} to prevent Retell retries on our DB failures
