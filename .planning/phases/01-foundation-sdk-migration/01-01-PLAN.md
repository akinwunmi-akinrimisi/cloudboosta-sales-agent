**003_rls.sql** -- Row Level Security policies (DATA-07):

Enable RLS on ALL 7 tables. Create policies:
- Anon role gets SELECT-only on all 7 tables (dashboard reads via anon key)
- No INSERT/UPDATE/DELETE policies for anon (dashboard cannot modify data)
- Service role bypasses RLS automatically in Supabase (no explicit policy needed)

Pattern for each table:
```sql
ALTER TABLE {table} ENABLE ROW LEVEL SECURITY;
CREATE POLICY "anon_read_{table}" ON {table}
  FOR SELECT TO anon USING (true);
```

Apply to: leads, call_logs, pipeline_logs, dial_schedules, programmes, pricing, objection_responses.

**005_functions.sql** -- Two trigger functions (DATA-03 + locked decision on DB-level enforcement):

1. **enforce_lead_status_transition()** -- BEFORE UPDATE trigger on leads:
   - Define valid_transitions as JSONB map matching EXACTLY the CONTEXT.md transitions:
     - new -> [queued, failed]
     - queued -> [calling]
     - calling -> [in_call, no_answer, voicemail, busy]
     - in_call -> [committed, follow_up, declined, not_qualified]
     - committed -> [payment_sent]
     - follow_up -> [queued]
     - no_answer -> [queued, declined]
     - voicemail -> [queued, declined]
     - busy -> [queued, declined]
   - ALWAYS allow any state -> do_not_contact (regulatory override per CONTEXT.md)
   - Skip if status unchanged (OLD.status = NEW.status)
   - RAISE EXCEPTION with descriptive message on invalid transition
   - Set NEW.updated_at = NOW() on valid transition
   - Create trigger: trg_enforce_lead_status ON leads BEFORE UPDATE OF status

2. **log_status_transition()** -- AFTER UPDATE trigger on leads:
   - If OLD.status IS DISTINCT FROM NEW.status, INSERT into pipeline_logs
   - Use component='status_transition', event='{old} -> {new}'
   - Store details as JSONB with old_status, new_status, triggered_by (using current_setting('app.triggered_by', true) which returns NULL if not set -- this is fine)
   - Create trigger: trg_log_status_transition ON leads AFTER UPDATE OF status

Both functions use LANGUAGE plpgsql. Include DROP FUNCTION IF EXISTS and DROP TRIGGER IF EXISTS before CREATE for idempotency.
