Create `execution/backend/migrate_phone_number.py` -- a one-time script to migrate the phone number from deprecated fields to weighted agents arrays.

The script must:
1. Load env vars via python-dotenv
2. Initialize Retell client from retell_config
3. Read RETELL_AGENT_ID from env (the agent to bind to the phone number)
4. Call `retell_client.phone_number.update()` with:
   - phone_number="+17404943597"
   - outbound_agents=[{"agent_id": agent_id, "weight": 1.0}]
   - inbound_agents=[{"agent_id": agent_id, "weight": 1.0}]
5. Print the response confirming:
   - Phone number
   - outbound_agents array (should show agent_id and weight)
   - inbound_agents array (should show agent_id and weight)
6. Verify by calling `retell_client.phone_number.get(phone_number="+17404943597")` and printing the result

Error handling:
- Catch retell.APIError and print the error details (status code, message)
- If RETELL_AGENT_ID is not set, print clear error message and exit
- If phone number not found, print helpful message about checking the number

Add a `--verify-only` flag (via sys.argv) that skips the update and only calls GET to check current configuration. This is safe to run multiple times.

Include a docstring at the top:
```
"""One-time migration: Update phone number +17404943597 to weighted agents format.

DEADLINE: March 31, 2026 -- deprecated fields stop working after this date.

Usage:
  python migrate_phone_number.py           # Run migration
  python migrate_phone_number.py --verify  # Check current config only

Requires: RETELL_API_KEY, RETELL_AGENT_ID in .env
"""
```

NOTE: Per RESEARCH.md Pitfall 2, weights MUST sum to 1.0. For single-agent, use weight=1.0.
