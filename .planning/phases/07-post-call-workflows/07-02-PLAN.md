1. **CSV Import Webhook** (n8n-nodes-base.webhook, typeVersion 2)

2. **Parse CSV** (n8n-nodes-base.extractFromFile, typeVersion 1)
   - operation: "csv"
   - binaryPropertyName: "data" (default binary property from webhook)
   - options: {} (use defaults -- auto-detect headers)

3. **Validate and Prepare Rows** (n8n-nodes-base.code, typeVersion 2)
   - JavaScript that processes ALL items from Parse CSV:
   ```javascript
   const E164_REGEX = /^\+[1-9]\d{6,14}$/;
   const valid = [];
   const errors = [];
   const seenPhones = new Set(); // In-batch dedup

   for (let i = 0; i < $input.all().length; i++) {
     const row = $input.all()[i].json;
     const rowNum = i + 2; // +2 for 1-indexed + header row
     const phone = (row.phone || '').trim();
     const name = (row.name || '').trim();

     if (!phone) {
       errors.push({ row: rowNum, reason: 'Missing phone' });
       continue;
     }
     if (!E164_REGEX.test(phone)) {
       errors.push({ row: rowNum, reason: `Invalid E.164 phone: ${phone}` });
       continue;
     }
     if (!name) {
       errors.push({ row: rowNum, reason: 'Missing name' });
       continue;
     }
     if (seenPhones.has(phone)) {
       errors.push({ row: rowNum, reason: `Duplicate phone in CSV: ${phone}` });
       continue;
     }
     seenPhones.add(phone);

     valid.push({
       phone,
       name,
       email: (row.email || '').trim() || null,
       location: (row.location || '').trim() || null,
       country: (row.country || '').trim() || null,
       status: 'new',
       source: 'csv_import',
     });
   }

   // Store errors and valid rows for downstream processing
   // Return one item with the full batch (SplitInBatches not needed for <1000 rows)
   return [{ json: { valid, errors, totalRows: $input.all().length } }];
   ```

4. **Insert Leads (Supabase RPC)** (n8n-nodes-base.httpRequest, typeVersion 4.2)
   - POST to `{{ $env.SUPABASE_URL }}/rest/v1/leads`
   - Headers:
     - apikey: {{ $env.SUPABASE_SERVICE_KEY }}
     - Authorization: Bearer {{ $env.SUPABASE_SERVICE_KEY }}
     - Content-Type: application/json
     - Prefer: resolution=ignore-duplicates,return=representation
   - Body: `{{ JSON.stringify($json.valid) }}`
   - The `Prefer: resolution=ignore-duplicates` header with the UNIQUE constraint on phone means duplicate phones are silently skipped and the response only contains successfully inserted rows.
   - options: { never_error: true, response: { response: { fullResponse: true } } }

   NOTE: If the n8n instance does not support the `resolution=ignore-duplicates` Prefer header (older PostgREST), use an alternative approach: POST each row individually in a Loop node and catch 409 Conflict responses. However, the bulk approach is preferred -- the Prefer header is standard PostgREST (which Supabase uses).

   ALTERNATIVE if Prefer header is unavailable: Use Code node to POST each row via fetch() and collect results:
   ```javascript
   const valid = $json.valid;
   const errors = $json.errors || [];
   let imported = 0;
   const supabaseUrl = $env.SUPABASE_URL;
   const serviceKey = $env.SUPABASE_SERVICE_KEY;

   for (const lead of valid) {
     const resp = await fetch(`${supabaseUrl}/rest/v1/leads`, {
       method: 'POST',
       headers: {
         'apikey': serviceKey,
         'Authorization': `Bearer ${serviceKey}`,
         'Content-Type': 'application/json',
         'Prefer': 'return=minimal'
       },
       body: JSON.stringify(lead)
     });
     if (resp.status === 201) {
       imported++;
     } else if (resp.status === 409) {
       errors.push({ row: lead.phone, reason: 'Duplicate phone in database' });
     } else {
       errors.push({ row: lead.phone, reason: `Insert failed: ${resp.status}` });
     }
   }
   return [{ json: { imported, skipped: valid.length - imported, errors, totalRows: valid.length + errors.length } }];
   ```

   Use the PREFERRED bulk approach (Prefer header) as primary. Include a comment in the JSON noting the alternative.

5. **Build Summary** (n8n-nodes-base.code, typeVersion 2)
   - Takes the Supabase response (inserted rows) and validation errors
   - Calculates:
     - `imported`: Number of rows in Supabase response body (successfully inserted)
     - `skipped`: Total valid rows minus imported (duplicates that were ignored)
     - `validation_errors`: Error array from Validate node
     - `total`: Total CSV rows processed
   ```javascript
   const insertResult = $json;
   const validationData = $('Validate and Prepare Rows').item.json;
   const insertedRows = Array.isArray(insertResult.body) ? insertResult.body : [];
   const validCount = validationData.valid ? validationData.valid.length : 0;
   const imported = insertedRows.length;
   const dbSkipped = validCount - imported;
   const validationErrors = validationData.errors || [];

   return [{
     json: {
       imported,
       skipped: validationErrors.length + dbSkipped,
       validation_errors: validationErrors,
       db_duplicates_skipped: dbSkipped,
       total_rows: validationData.totalRows || 0,
     }
   }];
   ```
