**Workflow structure (12 nodes):**

1. **Webhook Trigger** (n8n-nodes-base.webhook, typeVersion 2)
   - httpMethod: POST, path: "post-call"
   - responseMode: "onReceived" (respond immediately, process async)
   - Receives payload: {outcome, lead_id, programme_recommended, currency, lead_email, lead_name, follow_up_date}

2. **Route by Outcome** (n8n-nodes-base.switch, typeVersion 3)
   - mode: "rules"
   - 3 rules matching `{{ $json.body.outcome }}`:
     - Rule 0: equals "COMMITTED" -> output 0
     - Rule 1: equals "FOLLOW_UP" -> output 1
     - Rule 2: equals "DECLINED" -> output 2
   - fallbackOutput: "none" (drop unrecognized outcomes silently)

--- COMMITTED branch (output 0) ---

3. **Has Email? (Committed)** (n8n-nodes-base.if, typeVersion 2)
   - Check: `{{ $json.body.lead_email }}` is notEmpty
   - True -> Build Payment Email; False -> No-Op (skip, no email)

4. **Build Payment Email** (n8n-nodes-base.code, typeVersion 2)
   - JavaScript Code node that builds currency-specific HTML email body
   - Uses payload fields: lead_name, programme_recommended, currency
   - Bank details per currency (ONLY the lead's currency):

     - NGN: GTBank, Account 0631796979

   - Next steps: send proof to support@cloudboosta.co.uk, credentials in 24h, WhatsApp group, starts April 25 2026

   - Returns: { html, subject: "Your Cloudboosta Enrolment Details", reference }
   - Use the exact JavaScript from RESEARCH.md Pattern 5, enhanced with subject field

5. **Send Payment Email** (n8n-nodes-base.httpRequest, typeVersion 4.2)
   - POST https://api.mailersend.com/v1/email
   - Headers: Authorization: Bearer {{ $env.MAILERLITE_API_KEY }}, Content-Type: application/json

   - options: { never_error: true } (don't halt workflow on 4xx/5xx)

6. **Update to Payment Sent** (n8n-nodes-base.supabase, typeVersion 1)
   - operation: update, tableId: leads
   - matchingColumns: ["id"]
   - value: { id: lead_id from webhook, status: "payment_sent" }
