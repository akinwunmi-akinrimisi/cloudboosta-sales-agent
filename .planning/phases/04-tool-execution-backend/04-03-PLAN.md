Implementation:

1. Extract all args per tool_definitions.py schema:
   ```python
   outcome = args.get("outcome", "DECLINED")
   summary = args.get("summary", "")
   strategy = args.get("closing_strategy_used", "")
   persona = args.get("lead_persona", "")
   programme = args.get("programme_recommended", "")
   follow_up_date = args.get("follow_up_date")
   objections = args.get("objections_raised", "")
   motivation = args.get("motivation_strength", "")
   capacity = args.get("capacity_assessment", "")
   ```

2. Add logger.info at start:
   ```python
   logger.info("log_call_outcome called: outcome=%s strategy=%s persona=%s call_id=%s lead_id=%s",
               outcome, strategy, persona, call_id, lead_id)
   ```

3. Insert into call_logs FIRST (before lead update, so the row exists when the trigger fires):
   ```python
   supabase.table("call_logs").insert({
       "retell_call_id": call_id,
       "lead_id": lead_id,
       "outcome": outcome,
       "closing_strategy_used": strategy,
       "detected_persona": persona,
       "summary": summary,
   }).execute()
   ```
   NOTE: Other call_logs fields (transcript, recording_url, duration_seconds, call_cost, started_at, ended_at, from_number, to_number) are populated by the webhook call_ended/call_analyzed handlers in Phase 5. log_call_outcome only records what the LLM knows at end of conversation.

4. Update lead status based on outcome (only if lead_id is present and outcome is not NO_ANSWER):
   ```python
   if lead_id and outcome != "NO_ANSWER":
       status_map = {
           "COMMITTED": "committed",
           "FOLLOW_UP": "follow_up",
           "DECLINED": "declined",
           "NOT_QUALIFIED": "not_qualified",
       }
       new_status = status_map.get(outcome)
       if new_status:
           update_data = {
               "status": new_status,
               "last_strategy_used": strategy,
               "detected_persona": persona,
               "programme_recommended": programme,
               "outcome": outcome,
           }
           if outcome == "FOLLOW_UP" and follow_up_date:
               update_data["follow_up_at"] = follow_up_date
           supabase.table("leads").update(update_data).eq("id", lead_id).execute()
   ```
   The DB trigger (from Phase 1 005_functions.sql) automatically inserts into pipeline_logs when lead status changes.

5. Return success confirmation:
   ```python
   return {"result": f"Outcome '{outcome}' logged successfully"}
   ```
   This is the silent response -- speak_during_execution is False for log_call_outcome, so Sarah won't speak this text.
