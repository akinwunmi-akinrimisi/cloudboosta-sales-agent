Cal.com Booking Handler — Deep Dive

▼
1
Lead visits cal.srv1297445.hstgr.cloud/cloudboosta/cloudboosta-advisory-call
2
Fills in name, email, phone number (required — phone location type). Picks a 15-min slot.
3
Cal.com fires BOOKING_CREATED webhook to n8n at /webhook/cb-calcom-booking
4
Extract Attendee: Pulls phone from payload.location or attendees[0].phoneNumber
5
Match Lead: PostgREST query — or=(phone.eq.X,email.eq.Y). No match? Logs booking_no_lead_match.
6
Already Scheduled?: IF check — if status is already call_scheduled, logs double_booking_prevented
7
Atomic PATCH: PATCH /leads?id=eq.X&status=neq.call_scheduled — race condition guard. If 0 rows affected, another request won. Logs prevention.
8
Success: status set to call_scheduled, call_scheduled_at stored, booking_created logged

Race condition tested: 5 concurrent webhooks for the same lead results in exactly 1 booking_created and 4 prevented.

Call Flow — Deep Dive

▼
1
Greeting: "Hi [name], this is John calling from Cloudboosta." If returning lead, get_lead_context loads previous call history.
2
Discovery: Background, current role, goals, motivation. Natural conversation, not interrogation.
3
Gate 1 — Qualification: Profile A (no tech) / B (some tech) / C (junior cloud) / X (unclear). Determines programme recommendation.
4
Programme Lookup: lookup_programme returns pricing in lead's currency + matching testimonial.
5
Present Solution: Programme details, pricing, early-bird savings, success stories.
6
Objection Handling: Every objection triggers get_objection_response — 35+ types, multi-layer scripts, cultural nuances.
7
Closing: 6 strategies — doctor_frame, pain_close, inverse_close, nepq_sequence, diffusion, direct_close.
8
Email Collection: If no email on file, save_email captures it. WhatsApp fallback after 1 failed spelling attempt.
9
Advisory Call: If lead wants human advisor, check_advisor_availability queries Cal.com, confirms time, logs preferred_call_time.
10
Log Outcome: log_call_outcome records everything — outcome, persona, strategy, objections, summary, preferred time.
07

Post-Call Emails — Deep Dive

▼
Nurture

Personalised opening from call summary. Relevant career fact with urgency context (AI picks from 10). Upcoming webinar details. Brochure link. "Looking forward to having you onboard." Encourages cloud career pursuit.

Trigger: FOLLOW_UP (no date) or FOLLOW_UP + date (nurture_scheduled)
Door-Open

Genuine thanks. Respects decision. One career fact positioned casually. Brochure "for reference." No webinar push. "If things change, we're here." Zero pressure.

Trigger: DECLINED
Thank-You

Polite, brief (under 100 words). Thanks for their time. Wishes well. No brochure, no webinar, no programme push, no career facts. Just warmth.

Trigger: NOT_QUALIFIED
Missed-Call

"Tried to reach you." Brief Cloudboosta intro. Career fact for curiosity. Webinar as no-commitment option. Brochure + Cal.com booking link. Under 200 words.

Trigger: NO_ANSWER / VOICEMAIL / BUSY (first attempt only)
Payment

Programme-specific bank details (Revolut GBP/EUR/USD, GTBank NGN). Payment reference code. Instalment options. Next steps. Navy blue branded HTML template.

Trigger: COMMITTED (separate flow, not AI-generated)

AI engine: Claude generates each email from the call transcript. Picks 1 of 10 career facts matched to topics discussed. All sent via Resend from academy@cloudboosta.co.uk.

Retry Mechanism — Deep Dive

▼
1
Call disconnects with reason: no_answer, voicemail, or busy
2
Backend checks retry_count vs max_retries (default: 2)
3
If retries remain: retry_count += 1, next_retry_at = now + 60min, status back to queued
4
If retries exhausted: status set to declined (exhausted)
5
First no-answer only: If lead has email, missed-call email is sent (booking link + brochure + webinar)
